# 注释生成（Doxygen 中文）

## 适用场景

- 修改代码、重构、修复或优化时需要新增/更新注释。
- 新增函数、方法、接口或 API 时必须补充 Doxygen 中文注释。
- 函数内部逻辑变更仅在必要时添加少量行内或块注释。

## 核心规则

- 使用简洁中文描述“做了什么 + 为什么/约束”，避免复述代码。
- 新增函数必须补全 Doxygen 标签：`@brief`、`@param`、`@return`（无返回值写“无”），必要时添加 `@throws`/`@retval`。
- 修改既有函数的签名或行为时，同步更新对应注释。
- 函数内注释可选且少量：仅解释复杂逻辑、边界条件、算法步骤或非直观意图。

## Doxygen 中文模板

使用项目已有的 Doxygen 风格（`/** */` 或 `///`），并保持中文描述。

```c
/**
 * @brief 简要说明函数做什么
 * @param param1 参数含义与单位/范围
 * @param param2 参数含义与约束
 * @return 返回值含义（无返回值写“无”）
 * @throws XxxError 触发条件（如适用）
 */
```

## 示例

```c
/**
 * @brief 计算订单的应付金额
 * @param items 商品列表（不能为空）
 * @param discount 折扣比例，范围 [0, 1]
 * @return 应付金额（无折扣则为原价合计）
 */
```

## 执行流程

1. 识别新增函数、修改函数或函数内部逻辑变更。
2. 对新增函数补齐 Doxygen 中文注释；对已有注释按最新行为更新。
3. 仅在逻辑不直观时添加少量函数内注释，优先解释“为什么”。
