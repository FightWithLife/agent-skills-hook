# 图型分类

## 主图类型

| 类型 | 适用场景 | 输出要求 |
|------|----------|----------|
| 架构 / 边界图 | 需要说明系统长什么样、模块怎么分、边界在哪里 | 先写 `.puml`，再渲染成 `.md` ASCII |
| 接入点图 | 需要说明现有代码从哪里接入 | 接入点要编号，并在图下配小表 |
| 流程图 | 需要说明一个功能从输入到输出怎么走 | 保留主路径，再补关键分支 |
| 前后对照图 | 需要说明改动前后行为有什么变化 | 尽量复用相同节点名，方便比较 |
| 依赖拓扑图 | 需要说明模块依赖和布局关系 | 复杂布局优先 DOT |

## 补充图

只有在主图不够用的时候，才使用补充图：

- 时序图：顺序关系重要时
- 状态图：生命周期变化重要时
- 类图：数据结构形状重要时

补充图不能替代主图，只能补细节。

## 选择顺序

如果一时拿不准，按这个顺序选：

1. 架构 / 边界图
2. 接入点图
3. 流程图
4. 前后对照图
5. 依赖拓扑图
6. 补充图
