# 渲染规则

## 源文件和渲染文件

- `.puml` 是唯一源文件
- `.md` 是渲染结果
- 两个文件要同目录、同 basename
- 只要 `.puml` 变了，就必须从源文件重新整张渲染 `.md`
- 不要为了“修对齐”直接改渲染后的 ASCII

## 文件命名

- 文件名使用 `YY-MM-DD_name.puml`
- 文件名使用 `YY-MM-DD_name.md`
- 同一组图文档保持同名配对

## Markdown 排版

建议按这个顺序写：

1. 标题
2. 一句话说明用途
3. ASCII 图，放在 fenced code block 里
4. 必要时再补一小段说明或小表

## 可读性

- 标签要短，但必须有语义
- 一张图只回答一个问题
- 图一旦开始拥挤，就拆成多张
- ASCII 是默认输出
- SVG 只在确实有帮助时作为补充
