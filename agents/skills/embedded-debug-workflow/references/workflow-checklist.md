# 调试闭环检查清单

## 开始前

- 本轮要验证的现象是什么
- 预期看到什么日志、状态或返回值
- 是否需要刷写新固件
- 是否需要 USB 指令触发
- 若需要多条测试指令，是否已经按串行发送设计好顺序、判定条件和每条指令后的等待点
- 串口参数是否明确
- 若后续会刷写、USB 触发、重启或重新上电，是否已安排先开启串口持续抓取
- 输出目录是否明确为 `artifacts/`

## 执行中

1. 修改代码并补关键日志
2. 运行仓库构建命令
3. 确认固件产物路径
4. 先启动串口抓取，确认会话已开始落盘
5. 用刷写 skill 完成探测和刷写
6. 如有需要，按串行方式发送 USB 指令并读取响应；一条未完成前不得发送下一条
7. 等待现象出现或超时
8. 停止抓取并整理日志

## 收尾时必须回答

- 实际现象和预期一致吗
- 第一处异常出现在什么时间点
- 串口日志最后停在哪个关键状态
- 是否收到 USB 响应；若收到，内容是否符合预期
- 是否全程遵守单条指令串行发送；若没有，是否有用户明确授权
- 本轮是代码问题、环境问题、设备状态问题，还是证据不足
- 下一步最小动作是什么
