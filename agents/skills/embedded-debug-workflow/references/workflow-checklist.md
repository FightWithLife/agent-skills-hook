# 调试闭环检查清单

## 开始前

- 本轮要验证的现象是什么
- 预期看到什么日志、状态或返回值
- 是否需要刷写新固件
- 是否需要 USB 指令触发
- 串口参数是否明确
- 输出目录是否明确为 `artifacts/`

## 执行中

1. 修改代码并补关键日志
2. 运行仓库构建命令
3. 确认固件产物路径
4. 用刷写 skill 完成探测和刷写
5. 启动串口抓取
6. 如有需要，发送 USB 指令并读取响应
7. 等待现象出现或超时
8. 停止抓取并整理日志

## 收尾时必须回答

- 实际现象和预期一致吗
- 第一处异常出现在什么时间点
- 串口日志最后停在哪个关键状态
- 是否收到 USB 响应；若收到，内容是否符合预期
- 本轮是代码问题、环境问题、设备状态问题，还是证据不足
- 下一步最小动作是什么
