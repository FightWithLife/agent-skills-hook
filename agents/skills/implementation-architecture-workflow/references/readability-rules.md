# 可读性规则

## 命名

- 标签要短，且能直接看出含义
- 不要用 `step1`、`helper2` 这种占位名
- 前后对照图里的名字尽量保持一致

## 尺寸

- 一张图只回答一个问题
- 图开始拥挤时，直接拆图
- 小图多一点，通常比一张大而乱的图更好

## 源文件和渲染文件

- `.puml` 是源文件
- `.md` 是渲染结果
- 文件名使用 `YY-MM-DD_name.puml` 和 `YY-MM-DD_name.md`
- 不要手改 `.md` 来“修正”图形结构
- 只要 `.puml` 变了，就必须从源文件重新整张渲染 `.md`，不能手改局部字符；如果 `.md` 只放一张图就整份覆盖，如果还带正文就只替换图块

## Markdown 排版

建议按这个顺序写：

1. 标题
2. 一句话说明用途
3. ASCII 图，放在代码块里
4. 需要的话再补小表或简短说明

## 前后对照

- 尽量复用节点名
- 尽量让前后两张图长得像，方便比较
- 下方补一张变化摘要表，列清楚新增、删除、重排、改名
