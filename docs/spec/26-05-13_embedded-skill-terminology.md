# 嵌入式相关 Skill 术语清单

## 目标

统一 `embedded-debug-workflow` 及其配套 skill 的关键表述，减少同一约束的多种说法。本文只定义推荐说法，不引入新流程。

## 推荐术语

- `先开串口抓取`
  - 推荐含义：在任何可能触发复位、重启、重新枚举、模式切换或短时关键日志的动作之前，先启动持续串口抓取。
  - 推荐用法：刷写前先开串口抓取；USB 测试前先开串口抓取。

- `USB 指令串行发送`
  - 推荐含义：一次只发送一条指令；必须等待当前指令完成响应、超时或结果判定后，才能进入下一条。
  - 推荐用法：USB 指令默认严格串行发送。

- `日志优先`
  - 推荐含义：性能测试、耗时测试、功能联调默认先用日志获取证据，不默认切到 IO 打点或逻辑分析仪。
  - 推荐用法：性能/耗时测试默认优先走日志埋点。

- `IO 打点`
  - 推荐含义：通过拉高/拉低 IO 标记关键点位，再结合逻辑分析仪波形或导出 CSV 计算时间间隔。
  - 推荐用法：只有用户明确要求且已提供 IO 引脚映射时，才启用 IO 打点。

- `逻辑分析仪窗口经验文件`
  - 推荐含义：项目内持久化的抓取窗口经验文件，用于记录某类测试的建议窗口，避免重复把窗口设得过短或过长。
  - 推荐路径：`.agents/cache/logic_timing_windows.csv`

- `三证合一`
  - 推荐含义：串口日志、逻辑分析仪 CSV、触发记录三者要能相互对齐。
  - 推荐用法：抓取完成后必须做一次三证合一核对。

- `证据不足`
  - 推荐含义：已有日志或触发记录，但抓取窗口未覆盖完整关键沿，或关键证据之间无法对齐，不能直接下结论。
  - 推荐用法：串口有开始但 CSV 无起始沿，本次判定为证据不足。

- `窗口过短`
  - 推荐含义：逻辑分析仪采样结束时，目标流程仍在继续，或异常/超时路径没有被覆盖。
  - 推荐用法：若串口显示流程仍在继续而 CSV 已结束，本次判定为窗口过短。

## 使用约定

- 同一类约束尽量只保留一种主说法，避免在不同 skill 中反复换词。
- `SKILL.md` 使用短句直接写约束；长解释放到 `references/` 或专门的规范文档。
- 新增相关 skill 或 reference 时，优先复用本文术语。
