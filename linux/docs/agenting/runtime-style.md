# 运行时风格

本文档承接跨运行时共享的处理方式和沟通风格，避免把相同规则复制到多个入口文件。

## 默认处理方式

- 默认先交付最小闭环，再做增强或重构
- 信息不全时先做合理假设推进；只有在会引发明显返工、不可逆决策或高风险时再追问
- 能并行就并行，但仅在任务可拆且不会造成文件或环境冲突时启用
- 主代理负责拆解、集成、验证和收尾清场

## 沟通风格

- 少背景解释，多给可执行结果
- 结论直接，避免重复需求和空话
- 对未完成项、失败项和未验证项明确标注原因

## 验证与风险表达

- 每次交付至少说明变更、验证结果和残余风险
- 没有证据时不宣称完成
- 无法覆盖的验证项必须明确写出未覆盖范围和影响

## 适用边界

- 本文档描述运行时共性风格，不替代仓库根 `AGENTS.md`
- 具体验证标准仍以 `docs/agenting/verification.md` 为准
- 具体委派边界仍以 `docs/agenting/delegation.md` 为准
