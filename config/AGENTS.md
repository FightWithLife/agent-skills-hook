# agent-skills-hook 仓库总则

本仓库用于维护 Claude Code、Codex、OpenCode 和 Qoder 的协作提示词、配置和相关文档。

## 使用原则

- 入口文件只保留导航、边界和最小约束，不重复长规则
- 各运行时入口文件只保留对应增量，不把共享规则复制回入口
- 修改前先确认范围，尽量保持最小 diff
- 不改无关文件，不引入个人路径，不把运行时偏好写进共享入口
- 默认先交付最小闭环可复核结果，再做增强或重构
- 信息不全时先做合理假设推进；但不要假定用户已经完全厘清需求，遇到目标、范围或取舍不明确时，先向用户确认再执行

## 共享边界

- 适合本仓库的改动：提示词、配置、构建与验证说明、轻量配置模板
- 不适合在入口文件里展开的内容：具体检查清单、长流程、示例命令、深层协作规则
- 需要深度规则时，按主题拆分到 `docs/` 下属子目录，不在入口展开

## 产出要求

- 中文说明优先，术语保持一致
- 说明要能被各运行时共同引用
- 如果任务是 review，结论部分使用中文，并保留可追溯证据
- 超过一屏的细则默认下沉到 `docs/` 或对应运行时子目录
